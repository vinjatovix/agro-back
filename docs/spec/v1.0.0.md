status: frozen
type: architecture baseline

---

# 1. PURPOSE

This document defines the initial stable architectural specification of AgroApp.

It replaces any previous roadmap-style definitions and becomes the single source of truth for system architecture.

From this version onwards, all evolution MUST follow versioned spec increments.

---

# 2. SYSTEM OVERVIEW

AgroApp is a domain-driven backend system focused on agricultural modeling.

Core bounded contexts:

- Plant domain
- Bed domain
- Spatial system
- API layer
- Validation & error system
- Persistence layer

---

# 3. ARCHITECTURAL RULES

## 3.1 Immutability rule

Once a spec version is published:

- it MUST NOT be modified
- any change requires a new version (v1.1.0, v1.2.0, etc.)

---

## 3.2 Separation of concerns

The system is divided into:

- Domain (pure business logic)
- Application (use cases)
- Infrastructure (persistence, external systems)
- API layer (transport)
- Validation layer (input integrity)
- Documentation layer (OpenAPI)

---

## 3.3 Traceability rule

Every change MUST be traceable to:

- spec version
- epic module
- changelog entry
